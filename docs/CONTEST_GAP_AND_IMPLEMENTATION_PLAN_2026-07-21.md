# C 端主动式 AI 比赛差距与实现计划（2026-07-21）

## 1. 目标与时间事实

比赛命题要求面向元宝 C 端日常交互场景（搜索问答、写作、翻译、生图、文档总结等），做到“无需用户主动触发，AI 实时识别机会点或预判需求并主动提供服务”。用户提供的作品提交截止时间是 **2026-07-19 23:59:59**；本计划编写日为 **2026-07-21**，截止时间已过，但仍按可演示、可复测、可继续提交或答辩的质量完成产品。

本计划不恢复 FastAPI，也不自建底层数据库、定时器、对象存储或鉴权。业务逻辑继续运行在 EdgeOne Makers Agent/Function 上，状态继续使用 Makers Store/Blob/Conversation。

## 3. 尚未完全实现的比赛关键能力

### P0：必须先完成

1. **主动提醒的新鲜度与可操作性**
   - 左侧保留简洁提醒条，只展示未读或仍在稍后时段内的有效提醒。
   - 每条直接给“按建议处理 / 稍后 / 忽略”，设置和运行记录留在设置页。
   - 页面重新获得焦点、日程、地点/路线、偏好变化后事件驱动刷新，旧提醒必须被后端 reconciliation 淘汰。

2. **日程安全边界**
   - 前端：过去日期可查看，但不能添加、编辑、删除；日期输入最小值为今天。
   - 后端：普通网页调用、模型提案确认和腾讯会议同步都再次校验，禁止新增到过去或操作过去日程。
   - 模型：不存在的日程自然说明；时间重叠在确认卡明确警告；真实路线时间不足在保存后的主动提醒中立即出现。

3. **腾讯会议闭环**
   - 从腾讯会议 AI Skill 页面获取个人 Token，仅写入 EdgeOne Preview 环境变量。
   - 创建前展示主题、开始/结束时间并由用户确认。
   - 官方 MCP 成功后以 Action ID 做幂等键，只写入一条日历日程，保存 meeting_id、会议号、入会链接。
   - 重复确认不能重复创会或重复写日历。

4. **Cloudflare 视觉降级真实验证**
   - Preview 配置 Account ID；保留现有测试 Token 的 Preview-only 边界。
   - 分别验证多模态理解、文生图、图生图；记录 provider、延迟、错误码，不记录 Token。
   - 验证主 Provider 失败时只降级一次，不出现重复“正在检索/正在理解”。

5. **真实地点推荐验收**
   - 固定测试“推荐北京三里屯附近 3 家适合晚餐的餐馆并显示在地图”。
   - 每个地点必须同时有真实名称、地址、经纬度、provider/place_id；地图点位与文字数量一致。
   - rich_search 负责时效与口碑证据，位置服务负责实体真实性，任一缺失都不把地点写入地图。

### P1：提升比赛命题覆盖

1. 文档上传后主动建议“总结、提取待办、生成回复”，而不是只等用户再次提问。
2. 写作/翻译结果后根据上下文主动建议一种最有价值的改进（语气、长度、目标语言一致性），有冷却和每日上限。
3. 搜索回答后只在确有新信息价值时主动追踪，而不是重复检索或机械追问。
4. 生图后根据用户最近编辑行为主动提出一次具体改图建议，不暴露内部 Provider。
5. 建立机会排序：紧急安全风险 > 用户明确目标的下一步 > 截止时间 > 一般优化；同一机会有去重、冷却和过期。

### P2：演示与质量

1. 为上述每条能力补充可逐步点击执行的测试 Case、预期 UI、网络请求和持久化证据。
2. 记录“机会被发现 → 提醒生成 → 用户采纳/忽略 → 状态更新”的完整演示指标。
3. Preview 全量回归后再决定生产；本轮不发布生产。

## 4. 验收门槛

- 自动测试：Agent、Functions、前端、迁移脚本全部通过，构建和 lint 通过。
- 主动竞态：空白新对话可收到主动消息；用户先发消息时绝不被覆盖。
- 日程：过去日期三条路径均拒绝；重叠和路程不足都有自然提示；修改/删除后旧提醒消失。
- 地图：三里屯餐馆样例所有卡片都有真实位置标识，地图不刷新抖动。
- 腾讯会议：经用户确认创建一场明确标记的测试会议，MCP 返回成功且日历只出现一条。
- 视觉：Cloudflare 三种能力至少各有一次 Preview 真实调用证据，主密钥不进入 Git、日志或测试网站。
- 发布：只推送目标分支并创建 Preview，生产保持不变。

## 5. 执行顺序

1. 完成并测试主动提醒竞态、刷新与简洁侧栏。
2. 完成日程过去日期限制、重叠警告、路线时间回归。
3. 完成腾讯会议 Token 授权、真实调用与日历同步。
4. 使用已配置的 Preview-only Cloudflare 凭据完成三种能力的真实强制降级测试。
5. 完成三里屯真实餐馆地图回归。
6. 更新测试 Case、文档、GitHub 分支，部署新的 Preview 并执行全量验收。
