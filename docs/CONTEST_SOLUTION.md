# C 端主动式 AI 比赛方案

## 1. 命题解释

主办方要求面向搜索问答、写作、翻译、生图、文档总结等日常交互，实现“无需用户主动触发，AI 实时识别机会点或预判需求并主动提供服务”。本项目把它落成一个可验证的产品闭环，而不是在回答末尾机械追加“还需要什么”：

```text
用户行为 / 日程 / 文件 / Provider 事实
  → 机会识别（语义模型或可信业务事件）
  → Policy（价值、隐私、冷却、过期、免打扰、每日上限）
  → 持久 Event / Run / Notification
  → 左栏简洁提醒 + 下一空白对话的自然主动开场
  → 用户采纳 / 稍后 / 忽略
  → Makers Store 持久反馈与后续执行
```

AI 可以主动发现和表达机会，但不会越过确认边界修改日程、创建会议或执行其他高风险副作用。

## 2. 当前覆盖

| 场景 | 主动信号 | 无需用户再次提问时提供的服务 | 闭环 |
| --- | --- | --- | --- |
| 搜索问答 | 回答完成后的独立语义判断；只在信息会变化、有待跟进节点或重要未决问题时成立 | 提醒可在稍后检查更新；普通一次性搜索不骚扰 | 点击建议填入带上下文的追踪指令，手动发送后复用 rich search |
| 写作 | 识别已有草稿、明确受众、用途或交付节点 | 提出一种最有价值的语气、结构、长度或格式适配 | 用户采纳后在原上下文继续改写 |
| 翻译 | 识别术语一致性、目标受众、本地化或双语交付价值 | 提出术语复核或受众适配，不机械建议“再润色” | 用户采纳后继续处理原文本 |
| 生图 | 已生成图片且存在明确构图、尺寸或版本延展价值 | 提出一次具体可执行的改图建议 | 用户发送后进入现有图片版本链与 Makers Blob |
| 文档总结 | PDF/论文上传成功是可信 `file_uploaded` 信号 | 立即产生“文档已就绪”提醒；下一空白对话主动建议摘要、行动项或论文结论 | 用户采纳后打开现有阅读/总结链，文件与结果跨会话持久化 |
| 日程与地图 | 日程新增/修改/删除、天气、真实路线和时间冲突 | 即时刷新临近、冲突、恶劣天气和通勤不足提醒 | 通过确认 Action 修改后重新扫描，旧提醒失效 |
| 持续任务 | 模型提议、用户确认的 Workflow DAG | 到期步骤、失败、补偿和依赖阻塞主动提醒 | 用户完成/跳过/重试/补偿，Run 与反馈持久化 |

## 3. 机会识别原则

- 主动机会由独立 LLM 做语义判断，不使用关键词表替代意图理解。
- 一轮最多产生一条；置信度低于 `0.72` 丢弃。
- 简单问答、闲聊、算术、脑筋急转弯、一次性事实和已有待确认 Action 不生成机会。
- 语义机会同类型 6 小时冷却；每条有效期 1–168 小时；文档上传机会有效期 7 天。
- 过期、已读、忽略或未到稍后时间的机会不会进入左栏或新对话主动开场。
- 标题、正文和执行指令不得包含联系方式、精确地址、账号、证件、健康、财务、密钥等敏感信息。
- 机会识别失败、超时或返回无效 JSON 时静默跳过，不影响原回答。

## 4. 平台复用

| 基础能力 | 复用 EdgeOne Makers | 项目业务逻辑 |
| --- | --- | --- |
| 模型与流式运行 | AI Gateway、Agent Runtime | 机会判断提示词、价值阈值和公共 SSE 事件 |
| 状态 | LangGraph Store / Checkpointer | Event、Run、Notification、Workflow、Feedback 领域模型 |
| 会话主动开场 | Conversation Store | 只在空白会话写一条、竞态时用户消息优先 |
| 文件 | Makers Blob | 上传成功信号和文档下一步建议 |
| 定时唤醒 | EdgeOne Schedule + Cloud Function | 日程/工作流 Collector 和 Policy |
| 测试证据 | Makers Blob 预签名上传 | Case 结果、备注、主机审计和证据元数据 |

项目不自建数据库、对象存储、消息队列、模型服务、Cron 或通用 Trace。

## 5. 验收门槛

1. `PRO-09`：写作/翻译存在价值时实时产生最多一条机会；`1+1` 不产生。
2. `PRO-10`：上传 PDF 后无需第二条用户消息就出现文档提醒；下一空白会话自然主动开场。
3. `PRO-11`：同类型冷却、过期、稍后、采纳和忽略状态跨刷新有效。
4. 已有地图、日程、会议、生图、搜索、记忆和文档能力不得回退。
5. Python、Node、前端测试、lint 和生产构建全部通过；Preview 线上通过后才能升 Production。

逐步人工操作见生产站 `/test-cases/` 中的 69 条 Case。发布事实只写入 [`CURRENT_RELEASE.md`](CURRENT_RELEASE.md)。
