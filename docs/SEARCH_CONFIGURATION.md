# 联网搜索策略与设置

## 当前执行链

1. 独立 LLM 先读取当前问题和已过滤的非敏感长期记忆，决定是否联网、合并事实查询，并判断图片是否有助于理解。规划调用默认最多占用 6 秒（可用 `CAPABILITY_PLAN_TIMEOUT_SECONDS` 在 3–12 秒内调整）；超时时不使用关键词规则，主模型继续持有完整工具集并做语义决策。
2. 一轮对话只允许执行一次 `rich_search`；同一轮参数改写会复用同一个任务。
3. 只要 LLM 语义规划确认需要联网且图片上限不为 0，就允许从同一份事实结果提取并审核候选图；回答模型仍可不采用。规划器给出的事实与视觉意图合并进一次 WSA SearchPro 请求，不再为近义图片词额外调用第二次 SearchPro。
4. 来源页媒体可并发抽取，随后最多一次批量视觉调用审核真实像素；顺序为混元 → Cloudflare Workers AI → 百炼 Qwen-VL → Gemini。广告、二维码、Logo、UI、占位图和无关图不进入回答。
   WSA 新闻结果把主图放在独立图片字段或 `passage` 内嵌 `<img>` 时都会被识别；HTTP 主图统一升级为 HTTPS 后再送视觉审核，避免 Preview 的混合内容拦截。
5. 结果按规划器产出的合并查询、日期约束和搜索设置写入 LangGraph Store TTL 缓存；同义用户问法可以复用相同语义查询。Provider 超时且存在仍在安全陈旧窗口内的缓存时返回带 `stale_cache_hit` 标记的旧结果。

## 用户可调参数

右侧“主动策略与预算 → 联网搜索设置”持久化以下参数：

| 设置 | 可选值 | 默认 | 说明 |
| --- | --- | --- | --- |
| 网页结果上限 | 4 / 8 / 12 / 18 | 8 | 这是送入答案模型的去重结果上限，不是 Provider 计费配额 |
| 回答图片上限 | 0 / 2 / 4 | 2 | 0 表示关闭图片搜索和视觉审核 |
| 并行提取来源页图片 | 开/关 | 开 | 开启时并行抓取最多 4–6 个来源页；SearchPro 无论开关都只调用一次 |

默认 8/2/并行是当前推荐的平衡档：8 个来源足以支撑普通新闻综合；2 张图片能保持理解价值，同时把网页抓取页数和视觉审核候选控制在较小范围。深度档仍可提升到 18/4，但来源页串行提取可能触发 5 秒媒体预算，所以关闭并行只用于故障比较，不是推荐配置。

## 调用次数与性能证据

- 所有问题：每轮最多一次 `rich_search`，内部一次 WSA SearchPro；视觉词并入同一个 Query。
- 视觉审核：最多一次批量模型请求。
- 默认硬预算：能力规划 6 秒、SearchPro 10 秒、来源页媒体 5 秒、视觉审核 7 秒；因此未命中缓存的 `rich_search` 工具目标最坏约 22–24 秒，低于 30 秒。5–10 秒目标依赖 SearchPro 在预算内较快返回、结果自带主图或缓存命中；完整最终回答还包含主模型答案生成，不能把该时间全部算作 `rich_search`。
- Preview `dp2ak2xcj97a` 的真实取证：冷查询“最近AI有什么新进展”约 14.4 秒进入搜索、23.6 秒完成工具并开始组织，完整长回答约 48.3 秒；同一问题新会话复问时，持久缓存把外部检索压到约 1 秒，完整回答约 33 秒。两次均只执行一次 SearchPro，且新闻回答保留 1 张文章主图。
- 因此已经达到“`rich_search` 工具本身最慢低于 30 秒”的目标；5–10 秒可在缓存命中、Provider 快速返回或直接使用 SearchPro 主图时达到，但不能承诺所有冷查询在 5–10 秒内生成完整长答案。2026-07-21 的地点线上回归发现规划器可在主模型 300 秒网络超时前长期占住请求，现已把独立规划限制为 6 秒；若超时则由主模型继续语义路由，不取消 LLM 判断，也不增加搜索调用。

每次结果附带 `search_config`、`cache_hit`/`stale_cache_hit`、`cache_age_seconds`、`timings_ms` 和 `vision_diagnostics`，可以区分 Provider 请求数、缓存命中、搜索、网页媒体和视觉审核阶段。
