# 搜索、记忆与日程改造验证记录（2026-07-20）

本文记录 `agent/makers-native-persistence-fixes` 分支在本地 EdgeOne Makers 运行时中的验证结果。它不是生产发布记录；上线前仍需在 Makers Preview 使用相同用例复验。

## 1. 搜索规划与调用次数

- 搜索前保留独立 LLM 能力规划，不用关键词或确定性规则替代语义判断。
- 规划器同时接收经过隐私过滤的长期记忆，用于理解偏好、上下文与查询意图；敏感记忆不得进入外部搜索词。
- 一轮对话至多实际调用一次 `rich_search`。模型后续重复发起的同名工具调用会被抑制。
- 事实搜索与图片候选搜索继续并发；图片是否有帮助由规划模型判断。新闻、人物、地点、产品等适合视觉辅助的内容可以带图，简单算术和脑筋急转弯不强制搜索或配图。
- 图片候选改为一次多图视觉审核，不再逐图调用模型。
- 搜索结果按用户隔离持久化缓存；相同意图跨会话可以命中，时间敏感查询使用较短 TTL。

本地实测：

- `1+1 等于几？只给答案`：未调用搜索，直接回答 `2`。
- `请综合介绍最近人工智能领域的主要新进展`：只调用一次 `rich_search`；事实与图片并发，图片审核只有一次模型调用，搜索工具约 14.3 秒完成并返回 4 张相关图片。
- 同一汇率问题跨会话重复请求：第一次调用 Provider，第二次命中 Makers Store 缓存，没有再次调用 Provider。
- 前端忽略无工具名的流式伴随事件，不再把它渲染成第二次“正在检索”。

## 2. 自动记忆

- 稳定、长期、非敏感偏好由模型自动提取，无须用户逐条确认。
- 临时信息、低置信信息、过期信息和敏感信息会被过滤或清理。
- 用户界面只显示记忆数量、自动记忆开关和清空入口，不展示记忆原文。
- 搜索规划和回答模型都只能读取过滤后的有效记忆。

本地跨会话实测：

1. 在会话 A 输入长期偏好“回答先给结论，再给两条简短依据”。
2. `/intelligence` 返回 `memory_count: 1`，但 `memories` 为空数组，证明内容未对前端公开。
3. 在全新会话 B 询问“为什么天空是蓝色”，回答先给结论，再给两条简短依据。
4. 执行 `clear_memories` 后确认 `memory_count: 0`；本次测试数据已清理。

## 3. 自然语言日程 CRUD 与主动提醒

日程写操作仍通过确认卡执行，避免模型直接产生不可逆副作用。用户可以只用自然语言完成：

- 新增日程；
- 修改标题、描述、开始时间、结束时间或时长；
- 修改地点，地点必须先由 `search_places` 在真实地点库中核验；
- 删除日程；
- 对不存在或不唯一的日程得到自然、可执行的说明，不伪造日程 ID。

本地完整实测：

1. 通过对话创建带标题、描述、起止时间和故宫博物院地点的日程；地点带腾讯地点库返回的真实 `place_id`。
2. 通过对话将同一日程的标题、描述、起止时间和地点一次性改为新值，地点重新核验为天坛公园；原日程 ID 保持不变。
3. 将日程移入 24 小时窗口后生成主动提醒。
4. 只修改标题后，已有提醒内容原位刷新，没有产生重复提醒。
5. 修改一个不存在的日程时，模型自然说明未找到，并列出当前可匹配日程，没有创建伪造变更。
6. 删除日程后，关联提醒被标记为已失效，收件箱不再展示旧提醒。
7. 测试日程已全部删除。
