# 开发日志

## v4.2.0 — 图文并茂 + 死代码清理（2026-07-12）

### 改进项

1. **图片筛选改为视觉模型语义判断**
   - 视觉模型返回 `{"description": "...", "relevant": true/false}`
   - 只保留 `relevant=true`（与用户查询相关的图片）
   - 移除所有关键词正则过滤（`_USELESS_IMAGE_RE` 等 50+ 关键词）
   - 描述来自视觉模型（hy3），fallback 到搜索来源标题

2. **图片提取大幅扩大**
   - 网页范围从前 4 条 web 结果扩展到前 6 条所有结果（含百科）
   - 每页从提取 1 张（og:image）改为提取所有图片（og:image + img src + data-src），每页最多 10 张
   - 最终不再限制图片总数上限

3. **图片展示改为标准 Markdown**
   - 模型收到的是标准 Markdown 图片 `![描述](url)`，不再是自定义 `[[image:media-1]]` 标记
   - 模型自由决定在哪里插入，无需前端解析自定义标记
   - alt text 允许模型根据上下文改写，使描述与段落融为一体

4. **图文相关性改进**
   - 模型规则：只在与当前段落主题直接相关时才插入图片
   - 不相关的图片不用，不凑图
   - alt 文本可改写为上下文相关的描述

5. **搜索结果 snippet 清理**
   - 构建 context 时清除 `[[wsa] xxx]` 等搜索提供商标记
   - system prompt 增加"不在回答中提及搜索等内部过程"
   - 搜索结果质量过低时不注入 context

6. **视觉模型修复**
   - `hunyuan-vision` 模型名不存在，回退到 hy3（支持多模态输入）
   - 视觉描述失败时检测模型说"看不到图片"后返回空

### 死代码清理

- 删除 `agents/search/` 整个包（SearchAgent/SearchBundle/SearchRequest）— 被 skill 系统取代
- 删除 `agents/paper/` 整个包（PaperAgent/PaperBundle/PaperSearchRequest）— 无外部调用
- 删除 `search_service.py` 中的 `build_search_prompt()` — 仅被已删除的 SearchAgent 调用
- 删除 `image_skill.py` 中第一个被覆盖的重复 `confirmation_policy` 属性
- 更新 `image_skill.py` 的 `planner_steps`，移除废弃的 `ask_confirmation`
- 更新 `meeting_skill.py` 的 `output_modes`，移除废弃的 `confirmation_card`
- 修复 `richContent.test.ts` 测试断言匹配当前实现

|------|------|
| `backend/services/hunyuan_service.py` | 视觉模型返回 JSON（description + relevant），fallback 到 hy3 |

## v4.1.0 — UX 改进与 Bug 修复（2026-07-12）

### 改进项

1. **意图分类优化**
   - 意图分类 prompt 增加"询问信息 ≠ 要求执行"判断原则
   - "杭州有什么好玩的"路由到 chat 而非 travel
   - 意图分类请求注入当前时间，正确推算"明天""明晚"等相对日期

2. **ChatSkill 联网搜索增强**
   - 输入栏新增"联网搜索"复选框，默认勾选
   - 勾选时 ChatSkill 在回答前先执行搜索，将结果作为上下文注入 LLM
   - 搜索失败时静默降级为普通对话

3. **follow_ups 全覆盖**
   - BaseSkill 新增共享 `generate_follow_ups()` 方法
   - ChatSkill、SearchSkill（含零结果路径）、ImageSkill、MeetingSkill、TravelSkill 均生成追问
   - 追问通过 SkillStreamEvent.data.follow_ups 或 SkillResult.data.follow_ups 传递

4. **生图直接执行**
   - ImageSkill 改为 AUTO + side_effect=False
   - 不再返回确认卡片，直接调用混元 API 生成图片
   - 移除幂等限制，同一 prompt 每次生成不同图片

5. **会议直接创建 + 修改**
   - MeetingSkill 改为 AUTO + side_effect=False
   - 用户请求"创建会议"时直接调用 tmeet CLI 创建
   - 用户说"改到""调整"时先查询已有会议列表，找到匹配后调用 tmeet meeting update
   - MeetingService 新增 update_meeting() 和 list_meetings() 方法
   - 会议时间使用自然语言展示（"07月13日 15:00"），不暴露 ISO 格式

6. **搜索 404 过滤加强**
   - URL 有效性检查增加中文错误页面检测
   - 覆盖知乎"你似乎来到了没有知识存在的荒原"等 11 个中文 404 标记

7. **Policy 架构修复**
   - AgentPlan 新增 side_effect 和 trigger_origin 字段
   - Policy 只对 side_effect=True 的 HIGH risk AUTO 升级为确认
   - 非副作用技能（ImageSkill/MeetingSkill）直接执行，不再触发确认守卫崩溃

8. **System prompt 约束移除**
   - ChatSkill 移除"明确区分事实、推断和建议"等约束
   - 不再束缚大模型回答方式
