# V5 干净数据、Skills 与交互改造

本文是 `v5_20260722_clean` Preview 的实现说明和验收基线。目标是保留现有业务能力，同时让新版本从空状态开始；生产环境和旧数据不在本次 Preview 中修改。

## 1. 富媒体排版

- 搜索媒体不再作为固定的“文末图库”追加。
- 富搜索向主模型提供审核通过的媒体清单和 `[[YUANBAO_MEDIA]]` 占位协议；主模型根据叙述结构在适合的段落之间放置占位符，前端按顺序填入真实图片。
- 图片宽度为 AI 消息正文可用宽度的 `100%`，最大高度 `520px`，使用 `object-fit: cover`，不会再缩成 280px 小图。
- 没有对应图片的多余占位符会被移除；占位符不会显示给用户。消息恢复时，最终使用过的媒体元数据保存在 Makers 消息状态中，因此刷新页面后排版不丢失。
- 这套排版不增加搜索或视觉 Provider 调用次数；它只改变同一次回答中模型与前端的排版协作。

## 2. 丝滑微交互

所有原生和组件库可点击控件共用短反馈：`button`、链接、下拉摘要、单选/复选、开关、菜单、标签页、选择器、日期单元格和带 `role=button/switch` 的控件。悬停时轻微上移，按下时缩放到约 `0.97`，键盘聚焦显示柔和焦点环。

此外包括：

| 场景 | 动效 |
| --- | --- |
| 用户/AI 消息进入 | 280ms 淡入、7px 位移和轻微缩放 |
| 搜索图片、生成图、富媒体卡片出现 | 240–320ms 淡入，图片短暂去模糊 |
| 对话框、菜单、下拉层 | 短距离淡入和缩放 |
| 输入框、选择器聚焦 | 边框、阴影和焦点环过渡 |
| Skills 开关 | 滑块与底色同步过渡 |
| 日程编辑与删除确认 | 从对应日程条目下方 220ms 展开 |

系统开启“减弱动态效果”后，`prefers-reduced-motion` 会关闭位移、缩放、模糊、平滑滚动和非必要动画，不妨碍点击反馈与可访问性。

## 3. 日程就地操作

- 每条未来或今日的日程继续显示“编辑”和“删除”。过去日程仍是只读。
- 点击“编辑”后，编辑器直接在该日程下方展开，而不是跳到面板顶部；保存或关闭后就地收起。
- 点击“删除”后，使用同位置的内联确认条，不再调用浏览器原生确认框。
- 地图 Skill 关闭时仍可保存不含地点的日程；如果需要真实地点、地图或路线，编辑器显示“打开 Skills 广场”的上下文提示。

## 4. Skills 广场

当前业务能力被整理为 9 个产品 Skill，首次打开全部启用：

| Skill | 默认 | 依赖/联动 |
| --- | --- | --- |
| 核心对话 | 开，锁定 | 所有能力的基础 |
| 联网搜索 | 开 | 独立 |
| 视觉理解 | 开 | 搜索图片审核与图片理解 |
| AI 图片工作室 | 开 | 生成、编辑、版本管理 |
| 地图与真实地点 | 开 | 地点核验和路线 |
| 日程管理 | 开 | 地图是推荐依赖，不是硬依赖 |
| 主动式 Agent | 开 | 日程开启时才能提供完整日程提醒 |
| 论文与 PDF | 开 | 联网搜索是推荐依赖 |
| 腾讯会议 | 开 | 日程是硬依赖；个人 Token 决定是否可真实调用 |

硬依赖在开启 Skill 时自动补齐，例如开启腾讯会议会同时开启日程。推荐依赖不强制开启：当用户仅开启日程、关闭地图时，系统允许创建无地点日程，并在真正输入地点或请求路线时自然建议打开地图，而不是阻断整个日程功能。

开关同时约束主 Agent 工具目录和直接业务路由，避免仅隐藏前端却仍能从聊天绕过。主动式 Agent 开关还会同步主动提醒运行偏好。

## 5. 干净数据代际

本版采用逻辑代际 `v5_20260722_clean`，不是删除 Makers 托管数据：

- Conversation ID 使用 `yb5_` 短代际前缀与 24 位随机 ID，总长 28，满足 Makers 的 6–36 字符平台约束；浏览器会话键也使用新代际。
- LangGraph workspace、intelligence、proactive、message metadata、route cache 和 search cache 使用新 namespace。
- Blob 阅读索引、文件夹、设置、论文上传路径和新生成图片使用新代际路径。
- 会话列表只返回当前代际的对话。

因此新 Preview 第一次打开时没有旧会话、旧日程、旧记忆、旧主动提醒或旧阅读索引。旧对象仍由 Makers 保存，切回上一提交即可回滚；本次没有执行物理删除，也没有引入 SQLite 或自建数据库。

## 6. 验收

本地门槛：

```bash
.venv/bin/python -m unittest discover -s agents/_tests -p 'test_*.py'
npm --prefix frontend test -- --run
npm --prefix frontend run lint
npm --prefix frontend run build
npm test
python3 -m compileall -q agents
```

线上只创建 Preview，检查：空数据首次进入；AI 图片同正文宽且位于模型选择的段落；每类可点击控件都有短反馈；日程就地编辑/删除；Skills 默认全开、地图关闭时日程仍可用且出现依赖建议。上述全部通过前不得发布生产。
